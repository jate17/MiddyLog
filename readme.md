
# MiddyLog

> HL7 middleware log analyzer for LIS and blood gas instrument communication tracing.

MiddyLog is a command-line tool written in Rust for analyzing log files generated by healthcare middleware systems (such as RapidComm, POCcelerator). It parses HL7 v2.x message flows between a LIS (Laboratory Information System) and POCT instruments (e.g. blood gas analyzers), enabling fast troubleshooting without manually reading thousands of log lines.

---

## Features

- **Patient session trace** — extract all HL7 messages related to a specific patient ID across the full workflow (order → result → ACK)


---

## Supported Message Types

| Type | Direction | Description |
|------|-----------|-------------|
| `ORM^O01` | LIS → Middleware | New order |
| `ACK` | Middleware → LIS | Order acknowledgment |
| `ORU^R01` | Middleware → LIS | Result |
| `QRD` | LIS → Middleware | Patient query |
| `ERR` | Any | Error segment |

---

## Usage

```bash
# Analyze a single log file by patient ID
middylog --file Gateway-log.txt --pid 444

# Analyze multiple rotated log files
middylog --file Gateway-log.txt Gateway-log-1.txt Gateway-log-2.txt --pid 444
```

---


## Installation

```bash
git clone https://github.com/jate17/middylog
cd middylog
cargo build --release
```

Binary will be available at `./target/release/middylog`.

---

## Requirements

- Rust 1.75+
- Log files from HL7 v2.x compatible middleware (RapidComm, POCcelerator, or similar)
